---
name: spec
description: Interview user in-depth to create a detailed spec
---

Interview the user in depth about the requested project, feature, or change. Focus on non-obvious questions that clarify technical implementation, UX, constraints, tradeoffs, risks, dependencies, rollout, and success criteria.

Ask exactly one question at a time, then wait for the answer before asking the next one. Continue until the requirements are complete.

When you have enough information, write the spec to a file in the working tree. Include goals, scope, non-goals, requirements, constraints, open questions, and implementation notes.
